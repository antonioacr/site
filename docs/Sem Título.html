Quarto: Seu Escritório de Engenharia Digital para Criar Sites Incríveis 🏗️💻

E aí, futuro arquiteto digital! 👷‍♂️✨ Se você já sonhou em construir seu próprio site profissional, mas achou que precisava de um diploma em programação, tenho uma boa notícia: o Quarto é como ter uma equipe completa de engenheiros e arquitetos à sua disposição - só que em forma de software!

Pense comigo: assim como nenhum prédio surge do nada (primeiro vem a planta, o alicerce, a estrutura…), criar um site incrível também segue uma lógica de construção. E o Quarto? É exatamente o seu kit completo de ferramentas digitais para essa jornada!

Por que o Quarto é o “construtor” ideal para seu site? 🛠️

  • É como ter um projeto arquitetônico inteligente - ele já vem com a estrutura pronta, você só precisa decorar e personalizar

  • Suporta múltiplas linguagens (R, Python, Julia) - como ter especialistas em diferentes áreas da construção

  • Vai do simples ao complexo - desde um “sobrado digital” até um “arranha-céu” de site profissional

O melhor? Você não precisa ser um “engenheiro senior” em programação para começar. Se você já usou o RMarkdown (aquela “furadeira básica” da programação), vai se sentir em casa. O Quarto mantém o que já era bom e adiciona toda uma oficina de ferramentas profissionais!

Lançando os Fundamentos: Configurando o Projeto

Preparando o Terreno

  1. Abra o RStudio (sua central de construção).

  2. Crie um novo projeto:

    • Project: (None) > New Project... > New Directory > Quarto Website

  3. Defina o nome do projeto (seu “endereço digital”):

    • Para sites públicos: SeuUsuarioGitHub.github.io
      (Exemplo: joaosilva.github.io)

    • Para testes: meu-site-teste (nome temporário)

✅ Dica de Engenharia: Assim como um bom alicerce evita rachaduras, um nome bem definido evita retrabalhos futuros.

3. Estruturando o Site: A Planta Baixa Digital

3.1 Arquivos Essenciais (Elementos Estruturais)

Seu projeto inicial vem com os seguintes “componentes de construção”:

Arquivo Função Importante?
index.qmd Página principal (fachada do site) ⚠️ Não renomeie!
about.qmd Página “Sobre” (área institucional) Personalizável
_quarto.yml Projeto arquitetônico (configurações) Cérebro do site
styles.css Acabamentos visuais (cores, fontes) Opcional

Personalizando a Fachada (index.qmd)

  • Não renomeie! É obrigatório manter o nome index.qmd

  • Permite mistura de:

    • Markdown (formatação simples)

    • HTML (para elementos avançados)

    • Blocos de código executáveis (em R, Python, Julia)

    ---
title: "Meu Site Profissional"  # Nome do projeto
---

Dicas de Construção:

  • Adicione uma introdução (sua “placa de inauguração”).

  • Use blocos de código para funcionalidades dinâmicas (como “instalações elétricas” no site).

4. Acabamentos e Instalações: Personalizando o Site

4.1 Escolhendo o Tema Visual (Revestimentos)

No _quarto.yml, defina o estilo do site:

yaml

Copy

format:
  html:
    theme: minty  # Tema claro
    # Ou para modo dia/noite:
    theme:
      light: sandstone  # Dia
      dark: slate       # Noite
    css: styles.css     # Detalhes personalizados

4.2 Adicionando Redes Sociais (Sistemas de Comunicação)

Inclua links no YAML:

yaml

Copy

about:
  links:
    - icon: github
      href: https://github.com/seuusuario
    - icon: linkedin
      href: https://linkedin.com/in/seuperfil

5. Vistoria Final: Renderização e Publicação

5.1 Inspeção de Qualidade (quarto preview)

Antes de publicar, visualize o site localmente:

bash

Copy

quarto preview  # Abre uma prévia no navegador

5.2 Aprovação do Habite-se (GitHub Pages)

  1. Crie o arquivo .nojekyll (alvará de construção):

    bash

    Copy

    touch .nojekyll

  2. Configure o GitHub Pages:

    • Acesse Settings > Pages no repositório.

    • Defina Branch: main e Folder: /docs.

🌍 Seu site estará no ar em:
https://seuusuario.github.io

6. Manutenção e Expansões Futuras

Assim como um prédio precisa de reformas, seu site deve ser atualizado:

  • Adicione novas páginas (ampliações).

  • Atualize o conteúdo (reformas).

  • Sempre renderize antes de publicar (Build > Render Website).

🔧 Dica Final: Acompanhe o status na aba Actions do GitHub (seu “relatório de obras”).
